# CooNetwork README 设计文档

**日期**: 2026-07-20  
**类型**: 文档设计  
**状态**: 已完成

## 背景

CooNetwork 项目需要一份完整的中文 README 文档，帮助库使用者快速了解项目特性、安装方式和使用方法。

## 目标受众

**库使用者** — 其他 iOS/macOS 开发者想要集成 CooNetwork 到他们的项目中。

## 设计原则

1. **渐进式学习** — 从最简示例开始，逐步深入高级特性
2. **场景驱动** — 以实际使用场景组织内容，而非按类/协议罗列 API
3. **代码优先** — 每个概念都配完整的可运行示例
4. **架构透明但不强制** — 提到拦截器链、Actor 隔离等设计，但不深入细节
5. **快速导航** — 清晰的目录结构，使用者能快速找到需要的内容

## 文档结构

### 1. 项目标题和简介
- 项目名称 + 一句话介绍
- 核心价值主张：类型安全、Swift 6 并发、统一抽象
- 版本信息和平台支持
- Badges（版本、Swift、平台、License）

### 2. 核心特性（bullet points）
- Swift 6 并发安全（Actor 隔离）
- 类型安全的网络抽象
- 灵活的拦截器链
- 内置缓存、重试、去重
- 支持多后端（Alamofire、可扩展）
- 完善的错误处理

### 3. 快速开始
最简示例（5 行代码），展示定义请求 + 发起调用的完整流程。

### 4. 安装
- Swift Package Manager（推荐）
- CocoaPods

### 5. 基础使用
按使用频率排列：
- 定义请求（实现 `iAFRequest`）
- 发起请求（`request()` / `requestWithCache()`）
- 错误处理（捕获具体域错误）
- 响应验证（自定义验证规则）
- 取消请求

### 6. 常见场景
实用代码示例：
- 带参数的 GET/POST 请求
- 文件上传/下载
- 自定义请求头
- 使用缓存
- 进度监听
- 添加拦截器（日志、鉴权）

### 7. 高级特性
可选章节，面向需要深度定制的用户：
- 自定义拦截器
- 自定义响应解析
- 重试策略配置
- 去重机制

### 8. 示例项目
- 链接到 Examples/PodExample 和 Examples/SPMExample
- 提供运行说明

### 9. 架构概览
简要介绍双层设计、拦截器链、核心组件，引导到详细文档。

## 内容侧重

### 包含的内容
- ✅ 完整的代码示例（可直接运行）
- ✅ 常见使用场景的最佳实践
- ✅ 错误处理的推荐方式
- ✅ 安装配置步骤
- ✅ 架构的高层概览

### 不包含的内容
- ❌ 内部实现细节（如双层设计的技术实现）
- ❌ 设计决策的背景和权衡（留在 docs/design-decisions.md）
- ❌ 贡献指南和开发流程
- ❌ 详细的 API 文档（留给 DocC 或单独的 API 参考）
- ❌ License 和版权信息（已在 podspec 和 Package.swift 中声明）

## 示例代码风格

所有示例遵循以下原则：
1. **完整可运行** — 包含必要的 import 和类型定义
2. **贴近实际** — 使用真实的 API 端点结构（虽然是示例域名）
3. **注释适度** — 只在关键处添加注释，代码自解释
4. **类型明确** — 显式声明类型，帮助理解

## 语言和术语

- 使用简体中文编写
- 技术术语保留英文（如 Actor、Swift Package Manager）
- 代码注释使用中文
- 专有名词首次出现时提供英文对照

- [ ] 目录结构清晰，易于导航
- [ ] 从简单到复杂的学习路径明确
